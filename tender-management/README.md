# Tender Management System

B2B веб-приложение для управления тендерами и закупками с поддержкой множественных организаций и жесткой изоляцией данных.

## Описание продукта

Система предназначена для управления тендерами и закупками в рамках нескольких независимых организаций. Каждая организация имеет собственное изолированное пространство данных, пользователей и тендеров.

### Основные возможности

- **Управление тендерами**: создание, просмотр, редактирование, смена этапов/статусов
- **Совместная работа**: комментарии, уведомления о важных событиях в реальном времени
- **Ролевая модель**: три уровня доступа (администратор, менеджер, наблюдатель)
- **История изменений**: полный аудит всех изменений по тендерам
- **Мультитенантность**: полная изоляция данных между организациями

### Роли пользователей

| Роль | Права доступа |
|------|---------------|
| **admin** | Полный контроль над организацией: управление пользователями, всеми тендерами, настройками |
| **manager** | Операционное ведение тендеров: создание, редактирование, комментирование |
| **viewer** | Только чтение: просмотр тендеров и комментариев без права изменения |

## Технический стек

- **Backend**: Python 3.11, FastAPI, SQLAlchemy, PostgreSQL
- **Frontend**: TypeScript, React 18, Vite, Tailwind CSS
- **База данных**: PostgreSQL 15
- **Реалтайм**: WebSocket через FastAPI
- **Контейнеризация**: Docker, Docker Compose
- **Аутентификация**: JWT токены

## Быстрый старт

### Требования

- Docker и Docker Compose
- Git

```bash

После запуска создайте первого пользователя через API или используйте предоставленные тестовые данные (если настроены).

## Архитектура

### Мультитенантность

Приложение использует подход с изоляцией на уровне приложения:
- Каждая запись в базе данных привязана к организации через `tenant_id`
- Все запросы проверяют принадлежность данных к текущей организации пользователя
- Используются UUID для идентификаторов для дополнительной безопасности

### Безопасность

- **Аутентификация**: JWT токены с временем жизни 1 час
- **Авторизация**: Проверка прав доступа на основе ролей
- **Изоляция данных**: Строгая проверка tenant_id в каждом запросе
- **HTTPS**: Рекомендуется использовать в продакшене

## API Documentation

Полная документация API доступна по адресу: `http://localhost:8000/docs` после запуска приложения.

### Основные эндпоинты

- `POST /api/auth/login` - Аутентификация пользователя
- `GET /api/tenders` - Список тендеров организации
- `POST /api/tenders` - Создание нового тендера
- `GET /api/tenders/{id}` - Детали тендера
- `PUT /api/tenders/{id}` - Обновление тендера
- `POST /api/tenders/{id}/events` - Добавление события/комментария
- `GET /api/tenders/{id}/events` - История событий тендера
- `WS /ws/events` - WebSocket для уведомлений в реальном времени

## Разработка

```

### Frontend разработка

```bash
cd frontend
npm install
npm run dev
```
