# B2B Tender Management System

Веб-приложение для управления тендерами и закупками с поддержкой множественных организаций и жёсткой изоляцией данных.

## Продукт

### Что это
Система для совместной работы над тендерами в рамках организации с разграничением прав доступа, ведением истории изменений и уведомлениями о событиях.

### Пользователи и роли

| Роль | Описание | Права |
|------|----------|-------|
| `ORG_ADMIN` | Полный контроль по организации | Всё: управление пользователями, всеми тендерами, настройками организации |
| `TENDER_MANAGER` | Операционное ведение тендеров | Создание/редактирование тендеров, управление этапами, комментарии, просмотр всех тендеров организации |
| `VIEWER` | Только чтение | Просмотр тендеров и комментариев без права редактирования |

### Матрица прав

| Действие | ORG_ADMIN | TENDER_MANAGER | VIEWER |
|----------|-----------|----------------|--------|
| Просмотр тендеров организации | ✓ | ✓ | ✓ |
| Создание тендера | ✓ | ✓ | ✗ |
| Редактирование тендера | ✓ | ✓ | ✗ |
| Смена статуса/этапа | ✓ | ✓ | ✗ |
| Добавление комментариев | ✓ | ✓ | ✓ |
| Просмотр истории изменений | ✓ | ✓ | ✓ |
| Управление пользователями | ✓ | ✗ | ✗ |
| Удаление тендера | ✓ | ✗ | ✗ |

### Контекст закупок РФ
- Поддержка типов процедур: `44-ФЗ`, `223-ФЗ`, коммерческие закупки
- Поля, специфичные для госзакупок: НМЦК, номер извещения, площадка размещения
- Разделение данных между организациями — критическое требование

## Техническая архитектура

### Стек
- **Backend:** Python 3.11+, FastAPI, SQLAlchemy 2.0, Pydantic v2
- **Frontend:** React 18, TypeScript, Vite, TailwindCSS
- **БД:** PostgreSQL 15+
- **Realtime:** Server-Sent Events (SSE)
- **Аутентификация:** JWT (access + refresh токены)
- **Контейнеризация:** Docker, Docker Compose

### Модель данных

**Основные сущности:**
- `Organization` — организация-клиент
- `User` — пользователь с привязкой к организации и ролью
- `Tender` — тендер/закупка
- `TenderStage` — этап тендера
- `Comment` — комментарий к тендеру
- `AuditLog` — история изменений

**Изоляция данных:** Все запросы фильтруются по `organization_id` текущего пользователя. Реализовано через middleware и зависимости FastAPI.

### API

RESTful API с эндпоинтами:
- `/api/auth/*` — аутентификация
- `/api/organizations/*` — организации (только админ)
- `/api/users/*` — пользователи
- `/api/tenders/*` — тендеры
- `/api/comments/*` — комментарии
- `/api/events` — SSE поток уведомлений

## Быстрый старт

### Требования
- Docker 20+
- Docker Compose 2+
- **Опционально:** `make` (для удобных команд)

### Запуск

#### Вариант 1: С помощью Make (рекомендуется)

```bash
# Клонировать репозиторий
git clone <repo-url>
cd tender-management

# Запустить все сервисы
make start

# Просмотр доступных команд
make help
```

#### Вариант 2: Через docker-compose

```bash
# Клонировать репозиторий
git clone <repo-url>
cd tender-management

# Запустить все сервисы
docker-compose up --build

# Приложение доступно:
# Frontend: http://localhost:3000
# Backend API: http://localhost:8000
# API Docs: http://localhost:8000/docs
```

### Полезные команды

| Команда | Описание |
|---------|----------|
| `make start` | Запуск всех сервисов (сборка + фон) |
| `make stop` | Остановка всех сервисов |
| `make logs` | Просмотр логов в реальном времени |
| `make status` | Статус сервисов |
| `make health` | Проверка здоровья сервисов |
| `make clean` | Полная очистка (контейнеры, тома, образы) |
| `make dev-db` | Подключение к БД через psql |
| `make shell-backend` | Shell в контейнере backend |
| `make shell-frontend` | Shell в контейнере frontend |

## Что в MVP

✅ Реализовано:
- Регистрация организаций и пользователей
- Ролевая модель (ORG_ADMIN, TENDER_MANAGER, VIEWER)

- Комментарии к тендерам
- История изменений (аудит)
- Realtime уведомления через SSE
- Изоляция данных между организациями
- Аутентификация через JWT

- Экспорт в Excel/PDF
- Расширенная аналитика и дашборды
- Шаблоны тендеров

- Audit log экспортируемый

- GraphQL API

## Безопасность

- JWT токены с коротким временем жизни (15 мин access, 7 дней refresh)
- HTTPS в продакшене
- Пароли хешируются через bcrypt
- CORS настроен только на домен фронтенда
- SQL-инъекции предотвращены через ORM
- XSS предотвращён через React и Content-Security-Policy
- Изоляция организаций на уровне каждого запроса
